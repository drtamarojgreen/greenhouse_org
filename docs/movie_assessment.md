# Movie Character Face Architecture Assessment

## 1. Executive Summary
This report analyzes the recurring "adding/removing" cycle of facial features (eyes, pupils, and facial props) in the Movie 9 production pipeline. After investigation, the integrated mesh approach has been restored per user requirement.

## 2. Historical Analysis (Last 3 Months)
### 2.1 Key Event: Commit 5c9d797
*   **Title**: "Fix Redundant Eye Creation in Movie 9 Plant Characters"
*   **Action**: Removed eye and pupil geometry from `scripts/blender/movie/9/modeling/plant.py`.
*   **Context**: This commit was a reaction to "double-vision" bugs where characters appeared with two sets of eyes—one baked into the body mesh and one attached as independent props.

### 2.2 The Restoration
Following user feedback, the architectural decision made in commit `5c9d797` has been reversed. Facial features are once again integrated into the primary mesh generated by `PlantModeler`.

## 3. Technical Root Cause
### 3.1 Material Slot Mismatch
The `UniversalShader` and SDD audits (`Facial_Features.facts`) expect specific material slots:
*   Slot 0: Primary (Bark)
*   Slot 1: Secondary (Leaf)
*   Slot 2: Iris
*   Slot 3: Pupil

Restoring geometry to `PlantModeler` ensures these material indices are correctly assigned during the BMesh creation process.

## 4. Final Configuration

### 4.1 Integrated Mesh Approach (Active)
Facial features (eyes and pupils) are generated as part of the `PlantModeler.build_mesh` process. This ensures:
*   Immediate visual completeness in mesh-only previews.
*   Correct material index mapping for the `UniversalShader`.

### 4.2 Rigging Constraints
To prevent the "double-vision" bug, `PlantRigger` has been modified to deactivate modular prop creation for these features.

## 5. Verification Status
1.  **Restoration**: `PlantModeler.py` verified to contain `create_uvsphere` calls for eyes and pupils.
2.  **Redundancy Check**: `PlantRigger.py` verified to have `_create_minimal_face_props` disabled.
3.  **Audits**: `ProtagonistStructureAudit` passes for DYNAMIC:PLANT:SHADER configuration.
